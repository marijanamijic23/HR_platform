I think for me the hardest part when I started was how to connect the database with other parts of this project in the best way possible that I could find, but I liked this because this was the first time that I was working with a modern version of ASP.NET. On the faculty, we usually used .NET Framework. I used DbContext because it was a good way to make my implementation respect SOLID principles, and it was very interesting for me to use Migrations because it was the first time that I saw them, and I can say that they are a really good way to organize and control the database. I think it is really amazing how you can see the history of your database. Many-to-many relationships in CRUD operations with DbContext were a little hard in the beginning, but later I found them very interesting.
